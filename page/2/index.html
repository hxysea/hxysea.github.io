品读设计模式

写在前面的话

设计模式源起现实生活，譬如建筑设计师想要建造一栋建筑，他们会参照以往的图纸或者建造方法来绘制新建筑的图纸。将建筑师们所参照的这些图纸以及设计归纳总结乃至升华出来的一种针对某一类问题解决方案的设计思想与理念就是我们熟知的设计模式。

软件开发中的设计模式则是一套针对某一问题的一套设计良好的解决方案，是前辈们的心血与结晶。其优点有三：

  1. 可以大大缩短开发时间

    • 经过良好的设计以及实践检验
    • 是一套标准的解决方法
    • 针对类似的问题行之有效

  2. 减少开发成本

    • 提高代码可复用性
    • 提高健壮性
    • 提高可维护性

  3. 提高可交流性

    • 经过良好定义，易于理解和调试
    • 团队新成员更容易理解代码结构及上手

根据设计意图分为Creational（建造型）、Structral（结构型）、Behavioral（行为型）三种。使用设计模式的精髓在于学习与借鉴而不是一味的照搬，只有真正理解其设计意图，进而融汇贯通，方位真正领悟。

小酌Java数据类型

  1. 常量（不能改变的数值）

    1. 整数常量 –> 所有整数，如（1,2,3…）

      • 整数表现形式：

        • 十进制 0-9，满十进一
        • 八进制 0-7，满8进一，0开头（一个八进制位表示三个二进制位（不同0,1的排列组合)）
        • 十六进制 0-9，A-F，满16进一，0x开头（一个十六进制位表示四个二进制位（不同0,1的排列组合)）
        • 二进制 0-1

      • 自动类型提升

      • 强制类型转换
        PS:一个字节 = 8bit，就是八个二进制位，不同0,1的排列组合对应不同数据（映射对应），从而形成了编码表（如ascii表）

    2. 小数常量 –> 所有小数，如(1.1, 1.23,1.345…)

    3. 布尔常量 –> 只有两个值(true, false)
    4. 字符常量（数字和符号常量） –> 单个数字、字母或者符号用单引号标识，(‘a’, ‘b’, ‘*’, ‘1’, ‘2’, ‘3’)
    5. 字符串常量 –> （”hello world”, “a”, “”）
    6. null常量 –> 只有一个值（null）

Hexo & Markdown 2 get your Style

抓住刹那即逝的灵感，键盘敲打，自是一道风景。

　　所需不多，逐步操作，水到渠成。

  1. 登录Git官网，选择Windows版下载安装（版本选择最新，目前最新的为2.9.0）。
  2. 登录NodeJS官网，选择V4.4.7LTS版本下载安装。

  3. 本地安装Hexo，新建目录（如MyBlog），在目录下点击鼠标右键，选在Git bash，输入命令npm install -g hexo-cli

  4. 在Git bash下输入hexo version，校验Hexo安装是否成功。

  5. 初始化hexo环境，输入hexo init .，文件目录如图。

  6. 输入命令hexo server，启动服务，输入 http://localhost:4000 可访问首页。

  7. 修改_config.yml文件， 发布到GitHub上。

  8. 输入明林hexo generate和hexo deploy将博客内容发布到Github上。

Generate static files

Deploy to remote sites
