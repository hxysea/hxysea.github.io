3.抽象工厂设计模式

  • creational design pattern
design pattern

Abstract Factory Pattern

  1. What is it?

    抽象工厂模式是建立在工厂模式的基础上，对工厂模式中的工厂进一步抽象出一个抽象工厂。

  2. How to implement it?

    • 设计一个抽象工厂类，定义工厂的公共方法（获取对象）
    • 设计一个根据工厂类型获取对象的方法，去除原工厂模式中的if/else判断，使用抽象工厂类型作为入参，生产对象

  3. Code Demo
    ``` java
    package designpattern.creational.abstractfactory;

import designpattern.creational.factory.Computer;

/**

  • Created by AbuGe on 2016/11/21.
  • A.Abstract Factory
    */
    public interface ComputerAbstractFactory {
    public Computer createComputer();
    }

package designpattern.creational.abstractfactory;

import designpattern.creational.factory.Computer;
import designpattern.creational.factory.PC;

/**

  • Created by AbuGe on 2016/11/21.

  • B. Sub Class Factory
    */
    public class PCFactory implements ComputerAbstractFactory {

    private String ram;
    private String hdd;
    private String cpu;

    public PCFactory(String ram, String hdd, String cpu) {

    this.ram = ram;
this.hdd = hdd;
this.cpu = cpu;

    }

    @Override
    public Computer createComputer() {

    return new PC(ram, hdd, cpu);

    }
    }

package designpattern.creational.abstractfactory;

import designpattern.creational.factory.Computer;
import designpattern.creational.factory.Server;

/**

  • Created by AbuGe on 2016/11/21.
    */
    public class ServerFactory implements ComputerAbstractFactory {
    private String ram;
    private String hdd;
    private String cpu;

    public ServerFactory(String ram, String hdd, String cpu) {

    this.ram = ram;
this.hdd = hdd;
this.cpu = cpu;

    }

    @Override
    public Computer createComputer() {

    return new Server(ram,hdd,cpu);

    }
    }

package designpattern.creational.abstractfactory;

import designpattern.creational.factory.Computer;

/**

  • Created by AbuGe on 2016/11/21.
  • C. Fatory Class
    */
    public class ComputerFactory {
    public static Computer getComputer(ComputerAbstractFactory factory){
    return factory.createComputer();
    }
    }

package designpattern.creational.abstractfactory;

import designpattern.creational.factory.Computer;

Computer server = ComputerFactory.getComputer(new ServerFactory("32G", "1000G", "8GHZ"));
System.out.println("get pc config from abstract factory is " + pc);
System.out.println("get server config from abstract factory is " + server);

```
  1. Benefits
    • 面向接口编程，客户端与服务端解耦
    • 更易扩展，增加新工厂即可，与源代码无耦合
    • 更健壮，去除工厂模式的条件判断逻辑
