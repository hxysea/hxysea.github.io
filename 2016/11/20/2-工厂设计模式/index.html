2.工厂设计模式

  • creational design pattern
design pattern

Factory Design Pattern

  1. What is it?
    工厂是用来生成产品的，那么java中的工厂设计模式就是用来生产对象的。

    • 以往的设计是客户端需要什么对象直接new来创建，这样客户端和服务端就极度耦合在一起，需要知道服务端的所有实现。那么解耦则是优化设计的一种思路，抽象是解耦的一个方法。
    • 总结一下整条线就是：耦合–>解耦–>抽象。
    • 若需求中涉及的对象的关联关系是一个Super类和多个Sub类，且客户端需要根据参数获取不同的子类对象，那么根据此种需求设计一个工厂类来生产客户端所需对象就是对工厂设计模式的一个应用。

  2. How to implement it？

    • 一个Super类（可以是接口、抽象类或者普通类），抽象了多个子类的行为和属性
    • 多个Sub类，根据需求实现不同的行为
      -一个工厂类，根据不同的输入生成不同的子类对象

  3. Code Demo

  4. Benefits

    • 面向接口编程，给客户端提供的是接口而不是实现
    • 客户端与服务端具体实现解耦，更加健壮，更易扩展
    • 工厂类提供的接口与具体实现之间是继承关系
