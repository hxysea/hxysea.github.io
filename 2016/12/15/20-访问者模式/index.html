20.访问者模式

  • behavioral design patterns
design pattern

Visitor Pattern

  1. What is it?

    如果对于一组事物具有相似的操作，比如计算购物车里面的各种商品的总价。一种方法是每件商品包含计算价格的方法，缺点是计算方法与商品耦合。另一种方法是把操作逻辑抽象出来成为Visitor，物品使用该Visitor，则完成了解耦。这也就是访问者模式的精髓。

  2. How to implement it?
    • 设计操作对象接口及操作方法
    • 实现具体操作对象
    • 设计访问者接口
    • 实现具体的访问者接口

  3. Code Demo

    A.待操作对象

    B.访问者接口

    C.Test Program

  4. Benefit

    • 解耦，如果操作逻辑修改，只需修改访问者接口
    • 易于扩展，新增操作对象及逻辑，只需修改访问者接口，原有对象及操作不影响
    • 必须提前设计好操作的返回值类型，否则需要修改全部的访问者接口
    • 若访问者接口实现过多，则难以扩展
