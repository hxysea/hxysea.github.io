3.抽象工厂设计模式

Abstract Factory Pattern

  1. What is it?

    抽象工厂模式是建立在工厂模式的基础上，对工厂模式中的工厂进一步抽象出一个抽象工厂。

  2. How to implement it?

    • 设计一个抽象工厂类，定义工厂的公共方法（获取对象）
    • 设计一个根据工厂类型获取对象的方法，去除原工厂模式中的if/else判断，使用抽象工厂类型作为入参，生产对象

  3. Code Demo

2.工厂设计模式

Factory Design Pattern

  1. What is it?
    工厂是用来生成产品的，那么java中的工厂设计模式就是用来生产对象的。

    • 以往的设计是客户端需要什么对象直接new来创建，这样客户端和服务端就极度耦合在一起，需要知道服务端的所有实现。那么解耦则是优化设计的一种思路，抽象是解耦的一个方法。
    • 总结一下整条线就是：耦合–>解耦–>抽象。
    • 若需求中涉及的对象的关联关系是一个Super类和多个Sub类，且客户端需要根据参数获取不同的子类对象，那么根据此种需求设计一个工厂类来生产客户端所需对象就是对工厂设计模式的一个应用。

  2. How to implement it？

    • 一个Super类（可以是接口、抽象类或者普通类），抽象了多个子类的行为和属性
    • 多个Sub类，根据需求实现不同的行为
      -一个工厂类，根据不同的输入生成不同的子类对象

  3. Code Demo

  4. Benefits

    • 面向接口编程，给客户端提供的是接口而不是实现
    • 客户端与服务端具体实现解耦，更加健壮，更易扩展
    • 工厂类提供的接口与具体实现之间是继承关系

Creational Design Patterns

创建型设计模式

我们经常遇到的问题是创建对象方式未经设计或者增加设计的复杂度，Creational Design Patterns是针对特定特定需求下，如何实例化对象的一套优秀的设计方案或者说设计思想。

其中5个代表性的设计模式是：

  1. Singleton Pattern
  2. Factory Pattern
  3. Abastract Pattern
  4. Builder Pattern
  5. Prototype Pattern

这些设计模式用于通过创建对象的方式解决某一问题的，理解其思想，然后使用它。

1.说说单例那点事

Singleton Design Pattern

  1. What is it?
    单例也就是单实例，也就是JVM中只存在一个某类对象，且对外提供一个获取该对象的方法。

  2. Where you can use it?
    单例使用场景众多，如日志记录对象、JDBC驱动对象 、缓存对象、线程池对象等。在JDK的核心类库中， java.lang.Runtime, java.awt.Desktop等为单例对象，并且 AbstractFactory, Builder, Prototype, Facade等设计模式中也使用单例对象（使用单例模式的思想，也就是融会贯通）。

  3. How to implement it?

    • private constructor method

      • 禁止外部创建对象

    • private static this class instance variable

      • 存储该类唯一的对象

    • public method to get this class instance

      • 提供给外部获取该单例对象的方法

  4. Code Demo

  • Eager Initialization
    • Class加载时对象已经创建，缺点是无论Client是否使用，对象已经创建
    • 无异常处理
  • Static block initialization
    • 增加异常处理
  • Lazy Initialization
    • 优点是Client使用时才创建实例
    • 缺点是线程不安全
  • Thread Safe Sigleton
    • Client使用时创建
    • 线程安全
  • 双锁校验Thread Safe Singleton
    • 只有第一次创建时使用锁
  • Bill Pugh Singleton Implementation
    • 使用静态内部类，无需同步锁
    • 线程安全

  • Using Reflection to destroy Singleton Pattern

    • 通过反射可以破坏单例，创建新对象
  • Enum Singleton
    • 不灵活
    • 不支持懒加载
    • 解决反射破坏单例的问题
  • Serialization and Singleton
    • 反序列化时获取的是新对象
    • instanceOne hashCode = 17030099
    • instanceTwo hashCode = 3615232
  • 序列化破坏单例问题解决方法
    • 增加方法readResolve(反序列化时会调用)
    • 反序列化时读取的是单例对象不会重新创建新对象
    • instanceOne hashCode = 17030099
    • instanceTwo hashCode = 17030099

品读设计模式

写在前面的话

设计模式源起现实生活，譬如建筑设计师想要建造一栋建筑，他们会参照以往的图纸或者建造方法来绘制新建筑的图纸。将建筑师们所参照的这些图纸以及设计归纳总结乃至升华出来的一种针对某一类问题解决方案的设计思想与理念就是我们熟知的设计模式。

软件开发中的设计模式则是一套针对某一问题的一套设计良好的解决方案，是前辈们的心血与结晶。其优点有三：

  1. 可以大大缩短开发时间

    • 经过良好的设计以及实践检验
    • 是一套标准的解决方法
    • 针对类似的问题行之有效

  2. 减少开发成本

    • 提高代码可复用性
    • 提高健壮性
    • 提高可维护性

  3. 提高可交流性

    • 经过良好定义，易于理解和调试
    • 团队新成员更容易理解代码结构及上手

根据设计意图分为Creational（建造型）、Structral（结构型）、Behavioral（行为型）三种。使用设计模式的精髓在于学习与借鉴而不是一味的照搬，只有真正理解其设计意图，进而融汇贯通，方位真正领悟。
